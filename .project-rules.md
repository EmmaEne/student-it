# StudentIT Architecture & Development Rules

This document outlines the project structure and strict development rules to maintain a clean separation of concerns between Student and IT Coordinator (Admin) logic.

## 🏗 Project Structure

### `/core` (Shared Infrastructure)
- **`login.html`**: The unified entry point for all users.
- **`/assets`**: Global assets like logos, icons, and illustrations.
- **`/shared/css`**: Core design system, variables, and shared layout components.
- **`/shared/js`**: Coming soon... (Global utilities, auth engine, notification systems).

### `/portal-student` (Student Application)
- **`index.html`**: The main student interface.
- **`/js`**: Student-specific business logic and view rendering.
- **`/css`**: Student-specific styling overrides.

### `/portal-coordinator` (IT Coordinator Application)
- **`index.html`**: The main coordinator/supervisor interface.
- **`/js`**: Coordinator-specific business logic and view rendering.
- **`/css`**: Coordinator-specific styling overrides.

---

## 🚫 Strict Development Rules

1.  **Strict Isolation**:
    - Student components/scripts MUST NOT import or reference Coordinator-specific files.
    - Coordinator components/scripts MUST NOT import or reference Student-specific files.
    - Violations lead to "cross-role leakage" and fragile code.

2.  **Shared Logic Policy**:
    - Any logic or style used by BOTH roles MUST be placed in `/core/shared`.
    - Role-agnostic components (e.g., Toast, Loader, Theme Switcher) belong in `/core/shared`.

3.  **Naming Conventions**:
    - **Folders**: Kebab-case (e.g., `portal-student`, `shared-js`).
    - **JS Files**: Descriptive and role-prefixed (e.g., `student-views.js`, `coordinator-app.js`).
    - **CSS Classes**: Follow the established prefixing (e.g., `.class-card` for student vs `.status-card` for coordinator where possible).

4.  **No Tribal Knowledge**:
    - Every new feature must be placed in its respective portal folder.
    - If a feature is intended for both but implemented differently, create specific versions in each portal and share only the CSS variables from `/core`.

---

## 👨‍💻 For Junior Developers
*   **"Where does my code go?"**: If it's for students, put it in `portal-student`. If it's for staff, put it in `portal-coordinator`.
*   **"I want to change the primary blue color."**: Look in `core/shared/css/global.css` under `:root`.
*   **"I need a shared utility."**: Propose it for `core/shared/js`.
